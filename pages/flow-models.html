HTTP Request
klient
▼

    Header: Bearer <admin-token>
    Body: {{ first_name, last_name, email, password, role }}

2
AuthMiddleware → token ověření
src/Core/Auth.php
▼
  • 🔒Extrahuje Bearer token z HTTP_AUTHORIZATION
  • 🖄SELECT user JOIN user_token WHERE token=? AND expires_at>NOW() AND status='active'
  • 🔒requireRole('admin') – pouze admin může vytvářet uživatele
↓
3
UserController::store()
src/Controllers/UserController.php
▼
  • ⚙Validace: first_name, last_name povinné; email formát; password min 8 znaků

  • ⚙password_hash($password, PASSWORD_BCRYPT, ['cost'=>12])
  • 🖄INSERT INTO user (first_name, last_name, email, password, role, status, created_at)
  • ←HTTP 201 · {{ id, email, role }}
DELETE /users/:id – Soft-delete uživatele
1
UserController::destroy()
src/Controllers/UserController.php
▶
  • 🔒requireRole('admin')
  • 🖄SELECT id FROM user WHERE id=? AND status != 'deleted'
  • ⚙Nelze smazat sebe sama (Auth::id() === $id → HTTP 422)
  • 🖄UPDATE user SET status='deleted', deleted_at=NOW() WHERE id=? (soft delete)
  • ←HTTP 200 · {{ message: "User deleted" }}

PATCH /products/:id/stock – Úprava skladu (admin)
1
ProductController::adjustStock()
src/Controllers/ProductController.php
▼
  • 🔒requireRole('admin')
  • ⚙Body: {{ quantity: 5 }} – kladné přidá, záporné odebere
  • 🖄SELECT id, stock_quantity FROM product WHERE id=? AND deleted_at IS NULL
  • ⚙Nová hodnota = current + quantity; pokud < 0 → HTTP 422 "Insufficient stock"
  • 🖄UPDATE product SET stock_quantity=?, updated_at=NOW() WHERE id=?
  • ←HTTP 200 · {{ id, stock_quantity: <nová hodnota> }}
