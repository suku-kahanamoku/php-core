AuthMiddleware → Auth::check()
src/Middleware/AuthMiddleware.php · src/Core/Auth.php
▼
  • 🔒extractToken() – čte $_SERVER['HTTP_AUTHORIZATION']
    regex: /^Bearer\s+(\S+)$/i → token = "a3f9c2…"
  • 🖄SELECT u.id, u.email, u.role, u.first_name, u.last_name
    FROM user_token t JOIN user u ON u.id = t.user_id
    WHERE t.token = ? AND t.expires_at > NOW() AND u.status = "active"
  • 🔒Záznam nalezen → Auth::$currentUser = [id:1, role:'admin', …] uložen do statické cache
  • ⚙Token platný → pokračuje k controlleru

OrderController::updateStatus()
src/Controllers/OrderController.php
▼
  • 🔒Auth::requireRole('admin') – z cache, role = admin ✓
  • ⚙$id = 42, $newStatus = "cancelled"
  • ⚙Validace: "cancelled" ∈ povolených stavů ✓
  • 🖄SELECT id, status FROM `order` WHERE id = 42
    → status = "confirmed", záznam existuje ✓
  • ⚙Business rule: přechod confirmed → cancelled povolen ✓
  • 🖄UPDATE `order`
    SET `status` = 'cancelled', `updated_at` = NOW()
    WHERE id = 42
  • 🖄SELECT * FROM `order` WHERE id = 42 – načti aktuální stav pro response

Celková cesta každého requestu
HTTP request
  → Apache (.htaccess: mod_rewrite + Authorization passthrough)
  → bootstrap.php  (autoload + .env)
  → Request        (parse method / uri / body / headers)
  → Router         (match route + params)
  → CorsMiddleware (CORS hlavičky)
  → AuthMiddleware (ověření Bearer tokenu v DB)
  → Controller     (business logika + SQL)
  → Response       (json_encode + exit)
