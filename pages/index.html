⚡ Quick Start

1. Instalace & konfigurace
git clone <repo> && cd php-core
composer install
cp .env.example .env
# Upravte .env: DB_HOST, DB_NAME, DB_USER, DB_PASS
# FRANCHISE_CODES=default
2. Databáze & spuštění
mysql -u root -p -e "
  CREATE DATABASE php_core;
  CREATE USER 'admin'@'localhost'
    IDENTIFIED BY 'admin';
  GRANT ALL ON php_core.* TO 'admin'@'localhost';
"
php bin/migrate.php
php -S localhost:8000
3. První požadavek – login
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com",
       "password":"password"}'
# Vrátí token → uložte ho
4. Autentizovaný požadavek
TOKEN="váš_token_zde"
curl http://localhost:8000/api/products \
  -H "Authorization: Bearer $TOKEN"
# Spuštění testů:
php tests/api_test.php

💡 Klíčové koncepty

🏢 Multi-tenancy (franchise_code)

Každý HTTP požadavek je scoped na franchise_code, který se odvodí z HTTP Host headeru. Povolené kódy jsou v .env jako FRANCHISE_CODES=default,shop1. Neznámý host → 403.

Host: shop1.example.com → franchise_code = shop1
🔑 Bearer Token autentizace

Po přihlášení server vytvoří token (64 znaků) uložený v user_token tabulce. Token se posílá v hlavičce Authorization: Bearer <token>. Platí 24 hodin. Logout ho invaliduje na serveru.

Authorization: Bearer a3f9c2d8...
🧰 Dependency Injection (konstruktor)

Žádné statické třídy. Každý *Api a *Service dostává závislosti přes konstruktor. Entry point (api/module/index.php) vytvoří objekty a předá je dál.

$auth = new Auth($db);
$api  = new ProductApi($db, $code, $auth);
🏭 Modulární struktura

Každý modul má 3 vrstvy: *Repository (SQL), *Service (business logika + auth), *Api (routing + HTTP). Routy registruje metoda registerRoutes(Router $router).

Repository ← Service ← Api
📤 Response formát

Všechny odpovědi jsou JSON ve stejném formátu:

// Úspěch
{"success": true, "data": {...}}
// Chyba
{"success": false, "message": "..."}
👷 Role-based přístup

Tři výchozí role: admin, manager, user. Service volá $this->auth->require() nebo $this->auth->requireRole('admin'). Neautorizovaný přístup vrátí 401/403.

admin manager user
